Project Overview

I designed a Zelda-inspired browser puzzle game as a personal project to honor my favorite game series and to serve as the opening case study in my portfolio. My role included UX/UI design, prototyping, and front-end development using Figma and VS Code over a 1-week timeline.

Inspired by the iconic musical puzzles from The Legend of Zelda: Ocarina of Time, this project invites users to solve sequence-based puzzles using keyboard inputs, rewarding correct solutions with the progressive illumination of the Triforce symbol.

Interactive Demo

Click to start, then click the Triforce triangles. Use arrow keys to play the melody.

The Problem

Puzzle game fans lacked a web-based Zelda homage with intuitive controls and nostalgic visuals. Existing options were either too complex or didn't capture the Zelda spirit. Users wanted a simple, engaging puzzle experience reminiscent of classic Zelda dungeons.

Overly complex controls in similar games
Lack of Zelda-style puzzles online
Poor mobile support in existing games
"I just want a quick Zelda fix without downloading anything."
— User Insight from Research

Audio

N64 Ocarina Sound Synthesis

Custom audio synthesis using multiple oscillators, filters, and gain nodes to recreate the iconic N64 ocarina sound with authentic timbre and envelope shaping.

index.html - playNote() JavaScript 
function playNote(frequency) {
  const oscillator = audioContext.createOscillator();
  const oscillator2 = audioContext.createOscillator();
  const gainNode = audioContext.createGain();
  const filter = audioContext.createBiquadFilter();

  // Use square waves for N64 digital quality
  oscillator.frequency.value = frequency;
  oscillator.type = 'square';

  oscillator2.frequency.value = frequency * 2;
  oscillator2.type = 'sine'; // Soften the square wave

  // Lowpass filter for warmth
  filter.type = 'lowpass';
  filter.frequency.setValueAtTime(1800, audioContext.currentTime);
  filter.Q.setValueAtTime(2, audioContext.currentTime);

  // N64 ocarina envelope - quick attack, sustained
  masterGain.gain.setValueAtTime(0, audioContext.currentTime);
  masterGain.gain.linearRampToValueAtTime(0.35, audioContext.currentTime + 0.05);
  masterGain.gain.exponentialRampToValueAtTime(0.01, audioContext.currentTime + 0.8);
}

Animation

Triforce Canvas Animation

Dynamic Triforce animation with custom easing functions, gradient fills, and glow effects using the HTML5 Canvas 2D API and requestAnimationFrame.

index.html - redrawCanvas() JavaScript 
// Draw animating triangles on main canvas
for (let i = 0; i < 3; i++) {
  const startPos = savedTrianglePositions[i];
  const endPos = targetPositions[i];

  // Ease-out cubic interpolation
  const eased = 1 - Math.pow(1 - triforceAssemblyProgress, 3);
  const currentX = startPos.x + (endPos.x - startPos.x) * eased;
  const currentY = startPos.y + (endPos.y - startPos.y) * eased;

  ctx.save();
  ctx.translate(currentX, currentY);

  // Create rich gold gradient
  const gradient = ctx.createLinearGradient(0, -height/2, 0, height/2);
  gradient.addColorStop(0, '#FFE066');
  gradient.addColorStop(0.5, '#DAA520');
  gradient.addColorStop(1, '#8B6914');

  ctx.fillStyle = gradient;
  ctx.shadowColor = '#FFD700';
  ctx.shadowBlur = 12;
  ctx.fill();
  ctx.restore();
}

Testing & Iterations

Tested with 5 users (friends and family) to gather feedback and identify friction points before finalizing the design.

Issue Found

Unclear Puzzle Goal

Users struggled to understand what they were supposed to do when first landing on the puzzle.

Solution

Added Clear Instructions

Implemented an onboarding overlay with visual hints and a brief explanation of the controls.

Issue Found

Mobile Control Difficulty

Touch targets were too small and the virtual keyboard felt cramped on smaller screens.

Solution

Improved Mobile Controls

Increased button sizes by 40% and added more spacing between touch targets.

What I Learned

Clarity in onboarding is crucial

Users need to understand the goal immediately—don't assume they'll figure it out.

Early user testing saves time

Testing with just 5 users revealed critical issues that would have been costly to fix later.

Small animations boost engagement

Subtle feedback animations made the experience feel more polished and satisfying.
