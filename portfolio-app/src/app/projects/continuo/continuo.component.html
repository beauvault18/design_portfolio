CustomButton Component

A reusable, configurable button widget with hover animations, customizable styling, and consistent behavior across the app. Demonstrates proper widget composition and state management.

lib/widgets/custom_button.dart Dart 
class CustomButton extends StatefulWidget {
  final String text;
  final VoidCallback? onTap;
  final double width;
  final double height;
  final Color? backgroundColor;
  final double borderRadius;
  final double fontSize;

  const CustomButton({
    required this.text,
    this.width = 420,
    this.height = 72,
    this.borderRadius = 12,
  });
}

class _CustomButtonState extends State<CustomButton> {
  bool _hovering = false;

  @override
  Widget build(BuildContext context) {
    return MouseRegion(
      onEnter: (_) => setState(() => _hovering = true),
      onExit: (_) => setState(() => _hovering = false),
      child: AnimatedScale(
        scale: _hovering ? 1.05 : 1.0,
        duration: const Duration(milliseconds: 200),
        child: Container(
          decoration: BoxDecoration(
            color: _hovering ? bg.withOpacity(0.92) : bg,
            boxShadow: _hovering ? [BoxShadow(...)] : [],
          ),
        ),
      ),
    );
  }
}

Inactivity Timer Singleton

A singleton pattern implementation for managing app-wide inactivity detection. Essential for trade show kiosk mode‚Äîautomatically resets the app after periods of user inactivity.

lib/utils/inactivity_timer.dart Dart 
/// Singleton that manages global inactivity logic.
/// Triggers callback after [timeout] of inactivity.
class InactivityTimer with WidgetsBindingObserver {
  static final InactivityTimer _instance =
      InactivityTimer._internal();
  factory InactivityTimer() => _instance;
  InactivityTimer._internal();

  Timer? _timer;
  Duration timeout = const Duration(seconds: 60);
  VoidCallback? onTimeout;
  bool _paused = false;

  void start(VoidCallback onTimeout) {
    this.onTimeout = onTimeout;
    _reset();
    WidgetsBinding.instance.addObserver(this);
  }

  void userActivity() => _reset();

  @override
  void didChangeAppLifecycleState(AppLifecycleState state) {
    if (state == AppLifecycleState.paused) pause();
    else if (state == AppLifecycleState.resumed) resume();
  }
}

Custom Page Transitions

Smooth fade transitions between screens using AnimationController with custom easing curves. Creates a polished, professional feel throughout the navigation experience.

lib/screens/ai_in_healthcare/role_select_screen.dart Dart 
class _RoleSelectScreenState extends State<RoleSelectScreen>
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;
  late Animation<double> _fadeAnim;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      vsync: this,
      duration: const Duration(milliseconds: 600),
    );
    _fadeAnim = CurvedAnimation(
      parent: _controller,
      curve: Curves.easeInOut,
    );
  }

  void _onRoleTap(int idx) async {
    await _controller.reverse();
    await Navigator.of(context).push(
      PageRouteBuilder(
        pageBuilder: (context, anim1, anim2) =>
            RoleNewsScreen(role: roles[idx]),
        transitionsBuilder: (context, anim, _, child) =>
            FadeTransition(opacity: anim, child: child),
        transitionDuration: const Duration(milliseconds: 600),
      ),
    );
    _controller.forward();
  }
}

Project Overview

Continuo is a professional-grade iPad application, meticulously crafted with Flutter, that exemplifies the intersection of healthcare innovation and digital product design. The app is purpose-built for healthcare professionals‚Äîincluding nurses, doctors, and administrators‚Äîserving as a showcase for advanced AI-driven healthcare solutions at industry events, conferences, and business development meetings.

The primary objective is to deliver complex healthcare technology concepts through an intuitive, visually engaging, and highly interactive user experience. The app's design language is rooted in trust, clarity, and professionalism, aligning with the expectations of the healthcare sector.

Key Features & User Experience

iPad-Optimized, Responsive UI

Designed exclusively for landscape orientation, the interface leverages a grid-based layout, large touch targets, and adaptive components to ensure accessibility and ease of use in professional settings.

Interactive Video Demonstrations

A signature 50/50 split-screen design presents video content alongside contextual information, supporting four core product demos:

Transcription Intelligence

AI-powered real-time transcription of patient-provider conversations with smart note generation.

Bluetooth Vitals

Seamless integration with medical devices for automatic vital sign capture and documentation.

Voice Vitals

Hands-free vital sign entry through natural language voice commands.

Billing Codes

AI-assisted billing code suggestions based on clinical documentation and encounter data.

This format enhances comprehension and engagement, allowing users to explore features at their own pace.

Role-Based News Content

Personalized news carousels deliver relevant stories to users based on their professional role, utilizing color-coded cards and iconography for quick identification.

Contact Collection & Admin Dashboard

A seamless contact form captures user information, with data securely persisted locally. The admin dashboard, protected by a PIN, enables authorized export and management of contacts, supporting business development workflows.

Animated Transitions & Microinteractions

Custom fade transitions and subtle animations create a polished, modern feel, reinforcing brand identity and guiding user attention.

Inactivity Timer & Thank You Flow

An inactivity timer ensures privacy and resets the app to the main menu after periods of idleness. Upon contact submission, a branded thank you screen reinforces positive sentiment.

Professional Typography & Visual Language

The app employs SF Pro Display and a healthcare-appropriate color palette, with careful attention to hierarchy, contrast, and readability.
