Project Overview

I developed an AI-powered fall detection system designed for healthcare facilities and elderly care environments. The system uses YOLOv8 computer vision to analyze video feeds in real-time, detecting falls and immediately alerting on-duty staff to reduce emergency response times.

This project demonstrates the intersection of machine learning, full-stack development, and practical healthcare safety applications—combining Python backend services with a modern React frontend to create a complete, deployable solution.

The Problem

Falls are a leading cause of injury in healthcare and elderly care facilities. Current monitoring solutions rely heavily on manual observation or basic motion sensors that generate false alarms. Staff often learn about falls only after a patient has been on the ground for extended periods, increasing injury severity.

Delayed response times after falls occur
High false alarm rates with motion sensors
Understaffed facilities can't monitor all areas
"We need a system that can monitor common areas 24/7 and alert us the moment something happens—not after a patient has pressed their call button."
— Healthcare Facility Administrator

YOLOv8 Detection Pipeline

The detection pipeline processes images through YOLOv8, extracts bounding boxes, and draws annotated overlays with confidence scores and class-specific colors.

ml_service.py - detect_generic() Python 
def detect_generic(self, image: np.ndarray) -> List[Dict]:
    """Run YOLOv8 detection on image."""
    try:
        results = self.yolo_model(image, verbose=False)
        detections = []
        for result in results:
            for box in result.boxes:
                detections.append({
                    'label': result.names[int(box.cls)],
                    'confidence': float(box.conf),
                    'bbox': [int(x) for x in box.xyxy[0].tolist()]
                })
        return detections
    except Exception as e:
        logger.error(f"YOLO detection failed: {e}")
        return []

Key Features

Visual Detection with Bounding Boxes

Each detected object is highlighted with a color-coded bounding box and confidence percentage, making it easy to understand what the AI identified.

Users can drag images or videos directly onto the interface, with real-time upload progress and support for JPG, PNG, MP4, AVI, and MOV formats.

What I Learned

ML Model Integration Requires Robust Error Handling

Production ML services need extensive error handling—models can fail on unexpected inputs, and the system must gracefully degrade rather than crash.

User Context is Critical for Alerts

Detection alone isn't valuable—staff need location, timestamp, and clear instructions. The alert message design is as important as the detection accuracy.

Confidence Thresholds Require Tuning

The 70% default threshold balances false positives vs. missed detections. Real deployment would require iterative tuning based on facility feedback.
